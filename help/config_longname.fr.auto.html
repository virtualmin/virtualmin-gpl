Style de nom de domaine dans les noms d'utilisateur
Cette option détermine si le nom de domaine utilisé dans les noms d'utilisateur qui contiennent le domaine sera le nom de domaine complet, c'est-à-dire "virtualmin.com", juste la première partie, c'est-à-dire "virtualmin", ou le nom d'utilisateur/groupe pour le domaine. Par défaut, le nom du groupe est utilisé pour les nouveaux serveurs de niveau supérieur et la première partie du domaine est utilisée pour les sous-serveurs.

Dans certains environnements, il peut être plus facile pour les utilisateurs de comprendre un nom d'utilisateur comme « joeuser@virtualmin.com » pour leur nom de connexion POP/IMAP ou FTP que « joeuser.virtualmin ».

Ce champ détermine également comment le préfixe d'un nouveau serveur virtuel est calculé. Les options sont :


Nom de domaine complet
Le préfixe est toujours le même que le nom de domaine.

Nom d'utilisateur ou première partie du domaine
Le préfixe est le même que le nom de groupe Unix du serveur virtuel, s'il s'agit d'un serveur virtuel de niveau supérieur. Sinon, l'option suivante s'applique.

Première partie du domaine
Le préfixe est la première partie du domaine, sauf si un autre serveur virtuel utilise déjà ce préfixe. Ainsi, pour le domaine foo.com.au , le préfixe serait (par ordre de préférence) foo , foo.com ou foo.com.au .

Modèle personnalisé d'expression régulière
Le préfixe sera automatiquement généré en fonction d'un modèle d'expression régulière personnalisé, comme u-[0-9]{4} ou [az]-[0-9]{2} ou user-[0-9]{2}-${year}-${month}-${day} qui produira u-1234 ou x-89 ou user-99-2021-12-25 conséquence.



Il détermine également le nom d'utilisateur généré automatiquement pour un domaine de premier niveau, qui sera le même que le préfixe.
