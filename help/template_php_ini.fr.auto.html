Modèle de fichier de configuration PHP
Par défaut, lorsqu'un serveur virtuel exécute des scripts PHP en tant que propriétaire du serveur, Virtualmin copiera le fichier de configuration PHP global du système (généralement /etc/php.ini ) dans le répertoire ~/etc du domaine. Cela permet aux options PHP d'être définies différemment par serveur.

Ce paramètre peut être utilisé pour entrer un chemin différent pour le fichier de configuration PHP de base à copier. De plus, vous pouvez utiliser des substitutions Virtualmin comme ${DOM} dans ce fichier de modèle, qui sera remplacé lors de sa copie. Autres substitutions utiles disponibles:

${DOM} Le nom de domaine, tel que foo.com
${USER} La connexion d'administration du serveur, telle que foo
${HOME} Le répertoire personnel du serveur, comme /home/foo
${EMAILTO} L'adresse e-mail de contact du serveur virtuel
De plus, des blocs conditionnels tels que ${IF-MAIL} ... ${ENDIF-MAIL} et ${IF-WEB} ... ${ELSE-WEB} ... {$ENDIF-WEB} peuvent être utilisés.
