XML di autoconfigurazione di Thunderbird
Questo campo consente di personalizzare l'XML restituito ai client di posta che accedono al percorso URL /mail/config-v1.1.xml?emailaddress=user@example.com su un dominio utilizzando questo modello. I client come Thunderbird possono utilizzare questo XML per configurare automaticamente il nome host, il login, la porta, il protocollo e il nome utente del server di posta, in base all'indirizzo e-mail.

L'XML predefinito generato da Virtualmin funzionerà correttamente nella maggior parte dei casi, ma in alcune situazioni si desidera modificare i nomi dei server IMAP o SMTP o altre impostazioni. Nel modello è possibile utilizzare sostituzioni standard come ${DOM} e ${IP} , che verranno sostituite dal nome del dominio e dall'indirizzo IP quando vengono utilizzate.

Altre variabili che possono essere utilizzate nell'XML sono:

  • $SMTP_DOMAIN - Il nome di dominio, come example.com
  • $IMAP_HOST - Nome del server IMAP predefinito, come mail.example.com
  • $IMAP_PORT - Porta del server IMAP determinata automaticamente (come 143) prelevata dalla configurazione del server IMAP
  • $IMAP_TYPE - SSL o plain , a seconda della configurazione del server IMAP
  • $IMAP_ENC - Testo in password-cleartext o password-cleartext password-encrypted , a seconda della configurazione del server IMAP.
  • $SMTP_LOGIN - Nome utente completo per accedere ai server IMAP e SMTP, come bob.example
  • $SMTP_HOST - Nome del server SMTP predefinito, come mail.example.com
  • $SMTP_PORT - Porta del server SMTP determinata automaticamente (come 587) presa dalla configurazione del server di posta
  • $SMTP_TYPE - SSL o plain , a seconda della configurazione del server di posta
  • $SMTP_ENC : password-cleartext o STARTTLS , a seconda della configurazione del server di posta.
