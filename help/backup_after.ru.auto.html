Команда для запуска после резервного копирования
Если установлено, введенная команда или сценарий будет запускаться от имени пользователя root после выполнения резервного копирования по расписанию. Это может быть полезно для размонтирования съемных или удаленных файловых систем, в которые была записана резервная копия, или для очистки старых файлов резервных копий.

Команда может получить информацию о резервной копии через переменные среды, начинающиеся с BACKUP_ , например

BACKUP_DOMAIN_NAMES
Имена всех виртуальных серверов, для которых выполняется резервное копирование.

BACKUP_DEST
Целевой каталог или URL-адрес.

BACKUP_FEATURES
Имена всех функций Virtualmin, включенных в резервную копию. Если BACKUP_ALL установлен в 1, все функции включены.

BACKUP_STATUS
Установите 1, если резервное копирование прошло успешно, и 0, если нет.
