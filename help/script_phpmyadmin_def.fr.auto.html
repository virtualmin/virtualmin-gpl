Utiliser phpMyAdmin comme système par défaut
Une fois activée, cette installation de phpMyAdmin est définie comme installation par défaut pour l'ensemble du système. Un bouton « Gérer dans phpMyAdmin » s'affiche alors sur la page « Modifier les bases de données » pour les domaines ne disposant pas de leur propre installation de phpMyAdmin.

La version de phpMyAdmin utilisée dépend des éléments installés :

  • Si le sous-serveur possède sa propre installation de phpMyAdmin, c'est cette installation qui est utilisée.
  • Sinon, si phpMyAdmin est installé sur le serveur virtuel principal, cette installation sera utilisée à la fois pour le serveur principal et pour tous les sous-serveurs qui n'ont pas leur propre phpMyAdmin.
  • Sinon, l'installation par défaut de phpMyAdmin sera utilisée.

Cela peut s'avérer utile lorsque vous souhaitez une instance partagée de phpMyAdmin (par exemple, sous un nom d'hôte dédié comme pma.example.tld ) tout en autorisant des installations par serveur virtuel en cas de besoin.

Attention ! N’utilisez l’instance phpMyAdmin « par défaut » partagée sur un serveur virtuel dédié que si vous avez une confiance absolue dans le serveur virtuel qui l’héberge. Lorsque d’autres serveurs virtuels n’ont pas leur propre instance phpMyAdmin installée, ils utilisent cette instance par défaut, ce qui peut entraîner la transmission de leurs identifiants de connexion MySQL à cette instance. Si le serveur virtuel hébergeant l’instance est compromis, mal configuré ou contrôlé par une personne en qui vous n’avez pas une confiance totale, ces identifiants pourraient être exposés.
