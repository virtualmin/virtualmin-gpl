Nom de base de données par défaut
Virtualmin permet une création et des autorisations de base de données très flexibles. Ce champ vous permet de spécifier le nom par défaut des bases de données créées dans le cadre des domaines créés à partir de ce modèle. Par défaut, Virtualmin utilise la variable ${PREFIX}, qui est la première partie du nom de domaine, et sera généralement aussi le nom de connexion du propriétaire du domaine pour Webmin. Il peut être utile de changer cela, dans certaines circonstances, pour inclure un préfixe qui indique le modèle à partir duquel il a été généré, ou pour inclure des informations supplémentaires sur la base de données. Il est possible de combiner des variables et tous les caractères autorisés dans une base de données PostgreSQL et / ou MySQL (selon les bases de données disponibles).
