Opdracht uitvoeren na de back-up
Indien dit is ingesteld zal het ingevulde script of de opdracht worden uitgevoerd als root nadat de geplande backup is uitgevoerd. Dit kan erg handig zijn voor het ontkoppelen van verwijderbare of remote filesystemen waar de backup naar toe geschreven is, of voor het opschonen van oude backup files.

The command can discover information about the backup via environment variables starting with BACKUP_ , like

BACKUP_DOMAIN_NAMES
Names of all the virtual servers being backed up.

BACKUP_DEST
The destination directory or URL.

BACKUP_FEATURES
Names of all Virtualmin features included in the backup. If BACKUP_ALL is set to 1, all features are included.

BACKUP_STATUS
Set to 1 if the backup succeeded, 0 if not.
