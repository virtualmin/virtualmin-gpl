Comando para executar antes do backup
Se definido, o comando ou script inserido será executado como root antes que o backup seja executado conforme planejado. Isso pode ser útil para montar sistemas de arquivos removíveis ou remotos nos quais o backup deve ser gravado.

Se o comando falhar (com um status de saída diferente de zero), o restante do backup não continuará.

O comando pode descobrir informações sobre o backup por meio de variáveis de ambiente começando com BACKUP_ , como:

BACKUP_DOMAIN_NAMES
Nomes de todos os servidores virtuais cujo backup está sendo feito.

BACKUP_DEST
O diretório ou URL de destino.

BACKUP_FEATURES
Nomes de todos os recursos do Virtualmin incluídos no backup. Se BACKUP_ALL for definido como 1, todos os recursos serão incluídos.
