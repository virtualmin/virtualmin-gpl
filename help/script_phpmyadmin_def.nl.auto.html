Gebruik phpMyAdmin als systeemstandaard.
Wanneer deze optie is ingeschakeld, wordt deze phpMyAdmin-installatie ingesteld als de systeemwijde standaard. Op de pagina 'Databases bewerken' wordt dan een knop 'Beheren in phpMyAdmin' weergegeven voor domeinen die geen eigen phpMyAdmin-installatie hebben.

Welke phpMyAdmin wordt gebruikt, hangt af van wat er is geïnstalleerd:

  • Als de subserver een eigen phpMyAdmin-installatie heeft, wordt die installatie gebruikt.
  • Als de virtuele server op het hoogste niveau phpMyAdmin heeft geïnstalleerd, wordt die installatie gebruikt voor zowel de server op het hoogste niveau als voor alle subservers die geen eigen phpMyAdmin hebben.
  • Anders wordt de standaardinstallatie van phpMyAdmin gebruikt.

Dit kan handig zijn als u één gedeelde phpMyAdmin-instantie wilt (bijvoorbeeld onder een eigen hostnaam zoals pma.example.tld ), maar toch installaties per virtuele server wilt kunnen uitvoeren wanneer dat nodig is.

Let op: gebruik een gedeelde "systeemstandaard" phpMyAdmin onder een dedicated hostnaam alleen als u de virtuele server waarop deze draait volledig vertrouwt. Wanneer andere virtuele servers geen eigen phpMyAdmin hebben geïnstalleerd, gebruiken ze deze standaardinstantie. Dit kan betekenen dat hun MySQL-inloggegevens worden doorgegeven voor het inloggen. Als de hostende virtuele server gecompromitteerd, verkeerd geconfigureerd of beheerd wordt door iemand die u niet volledig vertrouwt, kunnen deze inloggegevens openbaar worden gemaakt.
