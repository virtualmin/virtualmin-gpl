Comando para executar após o backup
Se definido, o comando ou script inserido será executado como root depois que o backup for executado na programação. Isso pode ser útil para desmontar sistemas de arquivos removíveis ou remotos nos quais o backup foi gravado ou para limpar arquivos de backup antigos.

O comando pode descobrir informações sobre o backup por meio de variáveis de ambiente começando com BACKUP_ , como

BACKUP_DOMAIN_NAMES
Nomes de todos os servidores virtuais cujo backup está sendo feito.

BACKUP_DEST
O diretório ou URL de destino.

BACKUP_FEATURES
Nomes de todos os recursos do Virtualmin incluídos no backup. Se BACKUP_ALL for definido como 1, todos os recursos serão incluídos.

BACKUP_STATUS
Defina como 1 se o backup for bem-sucedido, 0 se não for.
