XML de configuration automatique d'Outlook
Ce champ vous permet de personnaliser le XML renvoyé aux clients de messagerie qui accèdent au chemin URL /autodiscover/autodiscover.xml sur un domaine à l'aide de ce modèle. Les clients comme Outlook peuvent utiliser ce XML pour configurer automatiquement le nom d'hôte, la connexion, le port, le protocole et le nom d'utilisateur du serveur de messagerie, en fonction de l'adresse e-mail.

Le XML par défaut généré par Virtualmin fonctionnera bien dans la plupart des cas, mais dans certaines situations, vous souhaitez modifier les noms de serveur IMAP ou SMTP ou d'autres paramètres. Dans le modèle, vous pouvez utiliser des substitutions standard comme ${DOM} et ${IP} , qui seront remplacées par le nom de domaine et l'adresse IP lors de l'utilisation.

Les autres variables pouvant être utilisées dans le XML sont:

  • $SMTP_DOMAIN - Le nom de domaine, comme example.com
  • $IMAP_HOST - Nom du serveur IMAP par défaut, comme mail.example.com
  • $IMAP_PORT - Port du serveur IMAP déterminé automatiquement (comme 143) pris dans la configuration du serveur IMAP
  • $IMAP_TYPE - Soit SSL ou plain , selon la configuration de votre serveur IMAP
  • $IMAP_SSL - yes ou no , selon la configuration de votre serveur IMAP
  • $IMAP_ENC - Soit en password-cleartext password-encrypted soit password-encrypted , selon la configuration du serveur IMAP.
  • $SMTP_LOGIN - Nom d'utilisateur complet pour se connecter aux serveurs IMAP et SMTP, comme bob.example
  • $SMTP_HOST - Nom du serveur SMTP par défaut, comme mail.example.com
  • $SMTP_PORT - Port du serveur SMTP déterminé automatiquement (comme 587) pris dans la configuration du serveur de messagerie
  • $SMTP_TYPE - Soit SSL ou plain , selon la configuration de votre serveur de messagerie
  • $SMTP_SSL - yes ou no , selon la configuration de votre serveur de messagerie
  • $SMTP_ENC - Soit password-cleartext ou STARTTLS , selon la configuration du serveur de messagerie.
