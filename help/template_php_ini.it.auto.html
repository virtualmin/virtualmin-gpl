File di configurazione modello PHP
Per impostazione predefinita, quando un server virtuale esegue script PHP come proprietario del server, Virtualmin copia il file di configurazione PHP globale del sistema (di solito /etc/php.ini ) nella directory ~/etc del dominio. Ciò consente di impostare le opzioni PHP in modo diverso in base al server.

Questa impostazione può essere utilizzata per inserire un percorso diverso per il file di configurazione PHP di base da copiare. Inoltre, è possibile utilizzare sostituzioni di Virtualmin come ${DOM} in questo file modello, che verrà sostituito quando viene copiato. Altre sostituzioni utili disponibili includono:

${DOM} Il nome di dominio, ad esempio foo.com
${USER} Il login di amministrazione del server, come foo
${HOME} La home directory del server, come /home/foo
${EMAILTO} L'indirizzo e-mail di contatto del server virtuale
Inoltre, è possibile utilizzare blocchi condizionali come ${IF-MAIL} ... ${ENDIF-MAIL} e ${IF-WEB} ... ${ELSE-WEB} ... {$ENDIF-WEB} .
