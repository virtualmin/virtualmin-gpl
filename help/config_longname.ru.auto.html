Стиль доменного имени в именах пользователей
Эта опция определяет, будет ли доменное имя, используемое в именах пользователей, содержащих домен, полным доменным именем, то есть "virtualmin.com", только первой частью, то есть "virtualmin", или именем пользователя / группы для домена. По умолчанию имя группы используется для новых серверов верхнего уровня, а первая часть домена используется для суб-серверов.

В некоторых средах пользователям может быть проще понять имя пользователя, такое как «joeuser@virtualmin.com» для своего имени входа POP / IMAP или FTP, чем «joeuser.virtualmin».

Это поле также определяет способ вычисления префикса нового виртуального сервера. Возможные варианты:


Полное доменное имя
Префикс всегда совпадает с именем домена.

Имя пользователя или первая часть домена
Префикс совпадает с именем группы Unix виртуального сервера, если это виртуальный сервер верхнего уровня. В противном случае применяется следующий вариант.

Первая часть домена
Префикс - это первая часть домена, если другой виртуальный сервер уже не использует этот префикс. Итак, для домена foo.com.au префиксом будет (в порядке предпочтения) foo , foo.com или foo.com.au

Пользовательский шаблон регулярного выражения
Префикс будет автоматически сгенерирован на основе шаблона пользовательского регулярного выражения, например u-[0-9]{4} или [az]-[0-9]{2} или user-[0-9]{2}-${year}-${month}-${day} что даст соответственно u-1234 или x-89 или user-99-2021-12-25 .



Он также определяет автоматически сгенерированное имя пользователя для домена верхнего уровня, которое будет таким же, как префикс.
