Comando da eseguire dopo il backup
Se impostato, il comando o lo script immesso verrà eseguito come root dopo che il backup viene eseguito secondo la pianificazione. Questo può essere utile per smontare i filesystem rimovibili o remoti su cui è stato scritto il backup o per pulire i vecchi file di backup.

Il comando può scoprire informazioni sul backup tramite variabili di ambiente che iniziano con BACKUP_ , come

BACKUP_DOMAIN_NAMES
Nomi di tutti i server virtuali di cui viene eseguito il backup.

BACKUP_DEST
La directory o l'URL di destinazione.

BACKUP_FEATURES
Nomi di tutte le funzionalità di Virtualmin incluse nel backup. Se BACKUP_ALL è impostato su 1, tutte le funzionalità sono incluse.

BACKUP_STATUS
Impostare a 1 se il backup è riuscito, 0 in caso contrario.
