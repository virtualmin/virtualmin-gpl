XML de configuración automática de Outlook
Este campo le permite personalizar el XML que se devuelve a los clientes de correo que acceden a la ruta URL /autodiscover/autodiscover.xml en un dominio utilizando esta plantilla. Los clientes como Outlook pueden usar este XML para configurar automáticamente el nombre de host, el inicio de sesión, el puerto, el protocolo y el nombre de usuario del servidor de correo, en función de la dirección de correo electrónico.

El XML predeterminado generado por Virtualmin funcionará bien en la mayoría de los casos, pero en algunas situaciones desea cambiar los nombres de servidor IMAP o SMTP u otras configuraciones. En la plantilla, puede usar sustituciones estándar como ${DOM} y ${IP} , que serán reemplazadas por el nombre del dominio y la dirección IP cuando se usen.

Otras variables que se pueden usar en el XML son:

  • $SMTP_DOMAIN : el nombre de dominio, como example.com
  • $IMAP_HOST - Nombre predeterminado del servidor IMAP, como mail.example.com
  • $IMAP_PORT : puerto del servidor IMAP determinado automáticamente (como 143) tomado de la configuración del servidor IMAP
  • $IMAP_TYPE : SSL o plain , según la configuración del servidor IMAP
  • $IMAP_SSL : yes o no , según la configuración del servidor IMAP
  • $IMAP_ENC : password-cleartext password-encrypted o password-encrypted , según la configuración del servidor IMAP.
  • $SMTP_LOGIN : nombre de usuario completo para iniciar sesión en los servidores IMAP y SMTP, como bob.example
  • $SMTP_HOST : nombre del servidor SMTP predeterminado, como mail.example.com
  • $SMTP_PORT : puerto del servidor SMTP determinado automáticamente (como 587) tomado de la configuración del servidor de correo
  • $SMTP_TYPE : SSL o plain , dependiendo de la configuración de su servidor de correo
  • $SMTP_SSL : yes o no , según la configuración del servidor de correo
  • $SMTP_ENC : password-cleartext o STARTTLS , según la configuración del servidor de correo.
