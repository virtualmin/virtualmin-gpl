Mal PHP-konfigurasjonsfil
Som standard, når en virtuell server kjører PHP-skript som serverens eier, vil Virtualmin kopiere systemets globale PHP-konfigurasjonsfil (vanligvis /etc/php.ini ) til domenets ~/etc katalog. Dette gjør det mulig å angi PHP-alternativer på en per-server-basis.

Denne innstillingen kan brukes til å angi en annen bane for den grunnleggende PHP-konfigurasjonsfilen som skal kopieres. I tillegg kan du bruke Virtualmin-erstatninger som ${DOM} i denne malfilen, som blir erstattet når den kopieres. Andre tilgjengelige erstatninger tilgjengelig inkluderer:

${DOM} Domenenavnet, for eksempel foo.com
${USER} Serverens administrasjonspålogging, for eksempel foo
${HOME} Serverens hjemmekatalog, som /home/foo
${EMAILTO} Den virtuelle serverens e-postadresse for kontakt
I tillegg kan betingede blokker som ${IF-MAIL} ... ${ENDIF-MAIL} og ${IF-WEB} ... ${ELSE-WEB} ... {$ENDIF-WEB} brukes.
