Als Systemstandard phpMyAdmin verwenden
Wenn diese Option aktiviert ist, wird diese phpMyAdmin-Installation als systemweite Standardinstallation festgelegt. Auf der Seite „Datenbanken bearbeiten“ wird dann für Domains ohne eigene phpMyAdmin-Installation die Schaltfläche „In phpMyAdmin verwalten“ angezeigt.

Welche phpMyAdmin-Version verwendet wird, hängt von der installierten Version ab:

  • Wenn der Subserver über eine eigene phpMyAdmin-Installation verfügt, wird diese Installation verwendet.
  • Andernfalls, falls auf dem virtuellen Server der obersten Ebene phpMyAdmin installiert ist, wird diese Installation sowohl für den Server der obersten Ebene als auch für alle Subserver verwendet, die keine eigene phpMyAdmin-Installation haben.
  • Andernfalls wird die standardmäßige phpMyAdmin-Installation dieses Systems verwendet.

Dies kann nützlich sein, wenn Sie eine gemeinsam genutzte phpMyAdmin-Instanz benötigen (zum Beispiel unter einem dedizierten Hostnamen wie pma.example.tld ), aber dennoch bei Bedarf Installationen pro virtuellem Server ermöglichen möchten.

Vorsicht! Verwenden Sie phpMyAdmin als Systemstandard unter einem dedizierten Hostnamen nur dann, wenn Sie dem virtuellen Server, auf dem es gehostet wird, uneingeschränkt vertrauen. Wenn auf anderen virtuellen Servern kein eigenes phpMyAdmin installiert ist, wird diese Standardinstanz verwendet. Dies kann bedeuten, dass die MySQL-Anmeldedaten für die Anmeldung an phpMyAdmin weitergegeben werden. Wenn der Host-Server kompromittiert, falsch konfiguriert oder von einer Person kontrolliert wird, der Sie nicht vollständig vertrauen, könnten diese Anmeldedaten offengelegt werden.
