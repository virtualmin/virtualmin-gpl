Mail client auto-configuration XML
This field allows you to customize the XML that is returned to mail clients that access the URL path /mail/config-v1.1.xml?emailaddress=user@example.com on a domain using this template. Clients like Thunderbird can use this XML to automatically configure the mail server hostname, login, port, protocol and username, based on the email address.

The default XML generated by Virtualmin will work fine in most cases, but in some situations you want to change the IMAP or SMTP server names, or other settings. In the template, you can use standard substitutions like ${DOM} and ${IP}, which will be replaced by the domain's name and IP address when used.

Other variables that can be used in the XML are :

  1. $SMTP_DOMAIN - The domain name, like example.com
  2. $IMAP_HOST - Default IMAP server name, like mail.example.com
  3. $IMAP_PORT - Automatically determined IMAP server port (like 143) taken from the IMAP server configuration
  4. $IMAP_TYPE - Either SSL or plain, depending on your IMAP server configuration
  5. $IMAP_ENC - Either password-cleartext or password-encrypted, depending on the IMAP server config.
  6. $SMTP_LOGIN - Full username to login to the IMAP and SMTP servers, like bob.example
  7. $SMTP_HOST - Default SMTP server name, like mail.example.com
  8. $SMTP_PORT - Automatically determined SMTP server port (like 589) taken from the mail server configuration
  9. $SMTP_TYPE - Either SSL or plain, depending on your mail server configuration
  10. $SMTP_ENC - Either password-cleartext or STARTTLS, depending on the mail server config.
