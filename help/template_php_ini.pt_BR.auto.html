Arquivo de configuração PHP do modelo
Por padrão, quando um servidor virtual executa scripts PHP como proprietário do servidor, o Virtualmin copia o arquivo de configuração PHP global do sistema (geralmente /etc/php.ini ) no diretório ~/etc do domínio. Isso permite que as opções do PHP sejam definidas de forma diferente por servidor.

Essa configuração pode ser usada para inserir um caminho diferente para o arquivo de configuração base do PHP copiar. Além disso, você pode usar substituições do Virtualmin como $ {DOM} neste arquivo de modelo, que será substituído quando for copiado. Outras substituições úteis disponíveis incluem:

${DOM} O nome do domínio, como foo.com
${USER} O login de administração do servidor, como foo
${HOME} O diretório inicial do servidor, como /home/foo
${EMAILTO} O endereço de email de contato do servidor virtual
Além disso, blocos condicionais como $ {IF-MAIL} ... $ {ENDIF-MAIL} e $ {IF-WEB} ... $ {ELSE-WEB} ... {$ ENDIF-WEB} podem ser usados.
