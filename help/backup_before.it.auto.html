Comando da eseguire prima del backup
Se impostato, il comando o lo script immesso verrà eseguito come root prima che il backup venga eseguito secondo la pianificazione. Questo può essere utile per montare file system rimovibili o remoti su cui scrivere il backup.

Se il comando fallisce (con uno stato di uscita diverso da zero), il resto del backup non procederà.

Il comando può scoprire informazioni sul backup tramite variabili di ambiente che iniziano con BACKUP_ , come:

BACKUP_DOMAIN_NAMES
Nomi di tutti i server virtuali di cui viene eseguito il backup.

BACKUP_DEST
La directory o l'URL di destinazione.

BACKUP_FEATURES
Nomi di tutte le funzionalità di Virtualmin incluse nel backup. Se BACKUP_ALL è impostato su 1, tutte le funzionalità sono incluse.
