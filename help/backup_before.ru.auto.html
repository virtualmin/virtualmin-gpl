Команда для запуска перед резервным копированием
Если установлено, введенная команда или скрипт будет запущена от имени пользователя root перед запуском резервного копирования по расписанию. Это может быть полезно для монтирования съемных или удаленных файловых систем, в которые должна быть записана резервная копия.

Если команда завершится неудачно (с ненулевым статусом выхода), остальная часть резервного копирования не будет продолжена.

Команда может получить информацию о резервной копии через переменные среды, начинающиеся с BACKUP_ , например:

BACKUP_DOMAIN_NAMES
Имена всех виртуальных серверов, для которых выполняется резервное копирование.

BACKUP_DEST
Целевой каталог или URL-адрес.

BACKUP_FEATURES
Имена всех функций Virtualmin, включенных в резервную копию. Если BACKUP_ALL установлен в 1, все функции включены.
