Comando para ejecutar antes de la copia de seguridad
Si se establece, el comando o script ingresado se ejecutará como root antes de que la copia de seguridad se ejecute según lo programado. Esto puede resultar útil para montar sistemas de archivos remotos o extraíbles en los que se debe escribir la copia de seguridad.

Si el comando falla (con un estado de salida distinto de cero), el resto de la copia de seguridad no continuará.

El comando puede descubrir información sobre la copia de seguridad a través de variables de entorno que comienzan con BACKUP_ , como:

BACKUP_DOMAIN_NAMES
Nombres de todos los servidores virtuales que se respaldan.

BACKUP_DEST
El directorio o URL de destino.

BACKUP_FEATURES
Nombres de todas las funciones de Virtualmin incluidas en la copia de seguridad. Si BACKUP_ALL se establece en 1, se incluyen todas las funciones.
