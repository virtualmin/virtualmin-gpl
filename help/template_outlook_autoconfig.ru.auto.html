Outlook авто-конфигурация XML
Это поле позволяет вам настроить XML, который возвращается почтовым клиентам, которые обращаются к URL-пути /autodiscover/autodiscover.xml в домене, используя этот шаблон. Клиенты, такие как Outlook, могут использовать этот XML для автоматической настройки имени хоста почтового сервера, имени входа, порта, протокола и имени пользователя на основе адреса электронной почты.

XML по умолчанию, сгенерированный Virtualmin, будет работать нормально в большинстве случаев, но в некоторых ситуациях вы хотите изменить имена серверов IMAP или SMTP или другие параметры. В шаблоне вы можете использовать стандартные замены, такие как ${DOM} и ${IP} , которые будут заменены именем домена и IP-адресом при использовании.

Другие переменные, которые можно использовать в XML:

  • $SMTP_DOMAIN - доменное имя, например example.com
  • $IMAP_HOST - имя сервера IMAP по умолчанию, например mail.example.com
  • $IMAP_PORT - автоматически определяемый порт сервера IMAP (например, 143), взятый из конфигурации сервера IMAP
  • $IMAP_TYPE - либо SSL либо plain , в зависимости от конфигурации вашего сервера IMAP.
  • $IMAP_SSL - либо yes либо no , в зависимости от конфигурации вашего сервера IMAP
  • $IMAP_ENC - либо открытый текстовый password-cleartext либо password-encrypted , в зависимости от конфигурации сервера IMAP.
  • $SMTP_LOGIN - полное имя пользователя для входа на серверы IMAP и SMTP, например, bob.example
  • $SMTP_HOST - имя SMTP-сервера по умолчанию, например mail.example.com
  • $SMTP_PORT - автоматически определяемый порт SMTP-сервера (например, 587), взятый из конфигурации почтового сервера.
  • $SMTP_TYPE - либо SSL либо plain , в зависимости от конфигурации вашего почтового сервера.
  • $SMTP_SSL - либо yes либо no , в зависимости от конфигурации вашего почтового сервера
  • $SMTP_ENC - либо открытый текстовый password-cleartext либо STARTTLS , в зависимости от конфигурации почтового сервера.
