Использовать в качестве системной настройки по умолчанию для phpMyAdmin.
При включении эта установка phpMyAdmin устанавливается в качестве системной по умолчанию. После этого на странице «Редактировать базы данных» для доменов, у которых нет собственной установки phpMyAdmin, появится кнопка «Управление в phpMyAdmin».

Какой именно phpMyAdmin будет использоваться, зависит от установленной операционной системы:

  • Если на подсервере установлена собственная версия phpMyAdmin, используется именно её установка.
  • В противном случае, если на виртуальном сервере верхнего уровня установлен phpMyAdmin, то эта установка используется как для сервера верхнего уровня, так и для любых подсерверов, у которых нет собственной установки.
  • В противном случае будет использоваться стандартная установка phpMyAdmin из этой системы.

Это может быть полезно, если вам нужен один общий экземпляр phpMyAdmin (например, под выделенным именем хоста, таким как pma.example.tld ), при этом сохраняя возможность установки для каждого виртуального сервера отдельно при необходимости.

Внимание! Используйте общий "системный экземпляр phpMyAdmin по умолчанию" под выделенным именем хоста только в том случае, если вы полностью доверяете виртуальному серверу, на котором он размещен. Если на других виртуальных серверах не установлен собственный phpMyAdmin, они будут использовать этот экземпляр по умолчанию, что может означать передачу их учетных данных MySQL для входа в систему. Если виртуальный сервер, на котором размещен сервер, скомпрометирован, неправильно настроен или контролируется кем-то, кому вы не полностью доверяете, эти учетные данные могут быть раскрыты.
