Stile del nome di dominio nei nomi utente
Questa opzione determina se il nome di dominio utilizzato nei nomi utente che contengono il dominio sarà il nome di dominio completo, ad esempio "virtualmin.com", solo la prima parte, ad esempio "virtualmin", o il nome utente/gruppo per il dominio. Per impostazione predefinita, il nome del gruppo viene utilizzato per i nuovi server di primo livello e la prima parte del dominio viene utilizzata per i server secondari.

In alcuni ambienti, potrebbe essere più semplice per gli utenti comprendere un nome utente come "joeuser@virtualmin.com" per il proprio nome di accesso POP/IMAP o FTP rispetto a "joeuser.virtualmin".

Questo campo determina anche come viene calcolato il prefisso di un nuovo server virtuale. Le opzioni sono:


Nome di dominio completo
Il prefisso è sempre lo stesso del nome a dominio.

Nome utente o prima parte del dominio
Il prefisso è lo stesso del nome del gruppo Unix del server virtuale, se si tratta di un server virtuale di livello superiore. In caso contrario, si applica la seguente opzione.

Prima parte del dominio
Il prefisso è la prima parte del dominio, a meno che un altro server virtuale non stia già utilizzando quel prefisso. Quindi, per il dominio foo.com.au , il prefisso sarebbe (in ordine di preferenza) foo , foo.com o foo.com.au .

Modello personalizzato di espressione regolare
Il prefisso verrà generato automaticamente in base al modello di espressione regolare personalizzato, come u-[0-9]{4} o [az]-[0-9]{2} o user-[0-9]{2}-${year}-${month}-${day} che produrrà u-1234 o x-89 o user-99-2021-12-25 conseguenza.



Determina inoltre il nome utente generato automaticamente per un dominio di primo livello, che sarà lo stesso del prefisso.
