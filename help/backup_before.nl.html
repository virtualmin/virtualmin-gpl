Opdracht uitvoeren vooraf de back-up
Indien dit is ingesteld zal het ingevulde script of de opdracht worden uitgevoerd als root voordat de geplande backup word uitgevoerd. Dit kan erg handig zijn voor het koppelen van verwijderbare of remote filesystemen waar de backup naar toe geschreven word.

Indien de opdracht mislukt (met een niet-nul afsluitwaarde) zal de rest van het backup proces niet doorgaan.

The command can discover information about the backup via environment variables starting with BACKUP_ , like:

BACKUP_DOMAIN_NAMES
Names of all the virtual servers being backed up.

BACKUP_DEST
The destination directory or URL.

BACKUP_FEATURES
Names of all Virtualmin features included in the backup. If BACKUP_ALL is set to 1, all features are included.
